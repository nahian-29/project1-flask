Talk abt file structure:

  • .github: explain git hub
    • workflows
      • dev.yml: making some actions seem automatic. This is the development version.
      • prod.yml: This is the same file as above but normally wouldn't include some tasks like tests. This is the file responsible for showing you this page now
  • app: this is the application files that are being used on the web application, some are being displayed, some are not.
    • static: these are files that doesn't change between pages. It can contain things like CSS files, JS, or images.
      • css
      • images
      • js
    • templates: this contains html templates that are used to display the webpages for you, the user to see.
      • cicdPage.html: CI/CD Page (next and last page)
      • dockerPage.html: Docker Page
      • gitPage.html: Git Page
      • index.html: Home Page
      • pythonPage.html: This is the page you're seeing
      • index-testing.html: This is the original page that I used to test Pytest, not used
    • __init__.py: This is the initation file for the application, this will create the app instance which contains the route for each page and the location of that page
    • run.py: This will start an instance of the initation file
  • calculator: This is the calculator portion that we are testing for another project, can be used to test pytest as well. Not used here so I won't be explaining much more.
  • tests: This is what PyTest uses to check whatever you want to check
    • __init__.py: This is empty but this is also the initation file for the test section
    • calculator_test.py: A test to check the results for the calculator section of the application
    • cicdPage_test.py: Test to check if the page has CI/CD as the result of the page
    • conftest.py: This creates the app instance and test if its functional
    • dockerpage_test.py: Test to check if the page has Docker as the result of the page
    • gitPage_test.py: Test to check if the page has Git as the result of the page
    • homepage_test.py: Test to check if the page has Home as the result of the page
    • pythonPage_test.py: Test to check if the page has Python - Flask as the result of the page
  • docker-compose.yml: This has the settings that the Dockerfile will use if accessed through a terminal
  • Dockerfile: This is the instructions for Docker to create a container for flask and this project
  • heroku.yml: Instructions for Heroku (the hosting site we are using for this project) and what to use to build
  • pytest.ini: This is the setting file for pytest, it has the directory for tests in case that you want to change the default
  • readme.md: This is simply a "How to" or "What is" file to this project
  • requirements.txt: This is for any pull requests and the libraries that the project will need to work properly
  • setup.py: This is a script to install the packages needed for the application mentioned

There are files like .gitignore and .dockerignore, these are files that for their respective services, will ignore directories or files. You can modify the contents if you like but not really used in this project.
