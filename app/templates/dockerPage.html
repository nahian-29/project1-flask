Docker is an interesting program. It works like a Virtual Machine (VM) without hard committing resources like VirtualBox or VMWare would. It provides containers, images and has the ability to share resources with your current machine when it needs it.

There are two terms that I need to explain in order to fully understand what Docker does:

Containers: are a storage space where an application (usually created from an image) can be virtualized and separate itself from the host computer and other containers. These are quick-and-dirty instances of a run-time environment without committing a lot of resources on virtualizing hardware like popular virtual machine applications.

Images: are like a starting operation installation file. These files are almost always read-only and used for creating multiple instances (containers). The best way to think about it is: the recipe of the cake is the image while each cake are the containers.

Docker has their own commands to operate within the terminal application (outside the Docker application itself). There are many but the ones that I know are simple to know and operate for the things that I would be using Docker for.

Here are the commands that I know in no particular order and the operation that they do:

  • docker ps and docker images: These are two commands that list the containers and the images (respectfully) created on your computer. It lists the ID and Names column that I will use for using the container in the terminal.
  • docker pull and docker push: This works like GitHub's pull and push. You can specify what content you are pushing to your Docker Hub repository. With pull, unlike with GitHub, you'll be pulling from the Docker Hub registry most of the time, meaning that you'll be pulling from a user/image[:TAG]
  • docker create: This allows you to create a container using an image file from Docker Hub or some DockerFile that you have in your directory.
  • docker run: This will start the container that you entered. It will create it if it doesn't exist and depending on the arguments that you pass through, you can detach and use a terminal to access the container.
  • docker exec: This will allow you to connect to your container that you have on your computer. You specify the container name and any arguments that you want to use. It may require you to use winpty before using any -t commands if the image itself doesn't have a dedicated terminal image.

And just like GitHub, there are repositories for each image you would create on Docker. The link to the project's repository is here: hub.docker.com/r/kirwinal/dockerflask.

And using your terminal application like Git, you can pull request this same application and begin creating your own version of this simple web application!

  1. Using your terminal application, you can use docker pull kirwinal/dockerflask:dev or docker pull kirwinal/dockerflask:master to download the project
  2. After pulling the image, you have to run and create the image using docker run -itdp [STARTING_PORT]:[ENDING_PORT] kirwinal/dockerflask:[TAG] TAG being dev or master
  3. To check both the name docker assigned and to see if you've created it, use docker ps and should see the list of containers that you have on your computer. Under the NAMES column, you should see the name that docker assigned the container.
  4. Then to finally access that container, use winpty docker exec -it [NAME] sh. This should finally allow you to use things like pytest which we will talk about in the next page.
