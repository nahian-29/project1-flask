Git is a terminal application that allows you to use any operating system as if it was Linux. It is very similar to Windows Powershell, and it allows you to use certain packages that Windows normally would not allow you to use. GitHub was created to have a hosting site for programmers and allows developers to share and have a common place to work on code as well while using git.

I'm going to explain some terminologies that are mainly used when talking about Git/GitHub.

Branch: is a different version of the same project. An example of a branch is like this project, there is a development branch where I tested a LOT of different changes with unused files for this project like the calculator files. There could be a lot of different branches for multiple purposes, it depends on the end goal of that branch. Click on the branch you want to see:

  • Live Branch <-- You're seeing this one
  • Development Branch

Merge: is taking the changes from one branch then adding them to the other, mainly the master branch. They usually start with a pull request (reviewing the changes from the main/master branch) then will go through with it once accepted.

Commit: is like the local "Save As" function. That will allow you to see what code changed, see which branch you're going to be working on, save the code with tags. You can commit and push meaning to do all of this and then move it to the main remote branch as working and usuable code.

Tags: are the notes written onto the commit to dictate what changed between the old version and the version you're going to push soon. You can also use this to dictate the important files of the project if you like, they could be added during, or after they are pushed.

Repositories: are like the storage container of the project. This will store all the project's files, branches, etc. This is what shared between users if they wanted to replicate the project. To see this project's repository: https://github.com/KirwinAl/flask
